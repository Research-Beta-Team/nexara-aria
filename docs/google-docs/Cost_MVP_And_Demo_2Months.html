Version: 1.0 (Model)  |  Audience: Board, Finance, Founders

Estimate for: (1) building the MVP in 1 month with 2 developers using vibe coding, and (2) running a 1-month demo with 20 potential customers using the product.

Developer rate: $200–500/month per developer.

Assumptions: 2 developers full-time; vibe coding = AI-assisted dev; MVP = core NEXARA + ARIA demo, auth, main flows, deployable for demos.

3. Month 2 — Demo to 20 Potential Customers

Developer rate: Same $200–500/month per developer. Assume 1 or 2 devs for support.

Assumptions: 20 potential customers have access and use the product (demos, ARIA). Not yet paying. Stable environment + light support.

5. What Drives the Range

  • Month 1: Developer monthly rate ($200 vs $500) and infra. Vibe coding does not change dev cost; it can reduce calendar time to MVP.
  • Month 2: Number of devs (1 vs 2), monthly rate, ARIA usage of the 20 demos, and whether you add dedicated demo/sales time.

6. After the Demo

If some of the 20 convert to paying customers, COGS (ARIA, support, infra share) apply per paying customer. See Cost to Serve 1 Customer. If you extend the demo or add more prospects, scale ARIA and infra with user count.

Developer rate used: $200–500/month per developer. Adjust ARIA usage and support level to your plan; replace with actuals after you run the 2-month cycle.
